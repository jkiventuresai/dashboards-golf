Certification Logic

1. Scorecard evidence
Card must be uploaded and readable.
2. Peer credibility
Friend group rates perceived honesty after the round.
3. Historical comparison
Submitted score is compared to recent scoring trend and handicap.
4. Decision
Approved, Review, or Denied. Any denial invalidates the group round.

MVP Build Prompt Used

This was the internal build instruction used to create the demo. In a real Codex/Replit build, the next version would add accounts, real uploads, OCR, audit trail, and group rules.

Build a self-contained MVP demo for a Golf Score Card Certification website. It should let a friend group upload scorecard evidence, rate each player's perceived honesty using a credibility score, compare the submitted round against each player's historical scoring pattern, and return an Approved or Denied certification. If any player is denied, the round is marked invalid for group records. Use static demo data only. No backend, no login, no external libraries, and no network calls. Include a dashboard, player certification cards, historical rounds, scoring logic explanation, and a mock upload/review panel.
Demo only. Uses static sample data and simulated certification logic. This is not an official handicap, GHIN, tournament, or USGA scoring system.
